# Frontend — Laufbahn

React 19 + Vite + TanStack Router (file-based) + TanStack Query + Jotai +
React Hook Form + Zod + Tailwind CSS v4 + shadcn-style UI.

## Layout

```
src/
├── routes/            file-based routes (TanStack Router)
│   ├── __root.tsx     router context (queryClient)
│   ├── login / register
│   └── _authed/       auth-guarded layout (AppShell) + pages
│       ├── applications/index.tsx   Kanban pipeline board
│       ├── applications/$id.tsx     detail with tabs
│       └── dashboard.tsx
├── features/          domain features (api hooks + components)
│   ├── auth, applications, interviews, contacts, follow-ups, activities
├── components/ui/     shadcn-style primitives (button, dialog, tabs, …)
└── lib/               api client, shared types, utils
```

## How it works

- **Auth** is cookie-based; the API client sends `credentials: 'include'`.
  `_authed` route `beforeLoad` checks `/auth/me` and redirects to `/login`.
- **Server state** lives in TanStack Query; mutations invalidate the relevant keys.
- **Pipeline board** groups applications by status into columns; drag a card to a
  valid next stage to trigger a status change (client mirrors the backend's
  transition rules to highlight valid drop targets).
- **Detail page** has tabs: Overview (edit/status/delete), Interviews, Contacts,
  Follow-ups, and an Activity timeline.

## Commands

```bash
pnpm dev        # vite dev server (proxies /api to VITE_API_URL)
pnpm build      # tsc -b && vite build
pnpm typecheck  # tsc -b
```

`routeTree.gen.ts` is generated by the TanStack Router Vite plugin on dev/build
and is git-ignored.
